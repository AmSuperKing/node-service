jQuery对象

当创建新元素(或选择现有元素)时，jQuery将返回集合中的元素。许多jQuery新手都认为这个集合是一个数组。它有一个索引为零的DOM元素序列、一些熟悉的数组函数和一个.length属性。实际上，jQuery对象要复杂得多。

DOM和DOM元素

文档对象模型（简称DOM）是HTML文档的表示形式。它可以包含任意数量的DOM元素。在较高的层次上，可以将DOM元素视为网页的“一部分”。它可能包含文本和/或其他DOM元素。DOM元件由类型描述，如<div>，<a>，或<p>，和任何数量的属性如src，href，class等。

元素具有与任何JavaScript对象一样的属性。在这些属性中，有属性如.tagName和方法如.appendChild()。这些属性是通过JavaScript与网页进行交互的唯一方法。

jQuery对象

事实证明，直接使用DOM元素可能很麻烦。jQuery对象定义了许多方法，以简化开发人员的体验。jQuery对象的一些好处包括：

兼容性 元素方法的实现因浏览器供应商和版本而异。以下代码段尝试设置存储在target中的<tr>元素的内部HTML：

xxxxxxxxxx
var target = document.getElementById( "target" );
target.innerHTML = "<td>Hello <b>World</b>!</td>";

这在许多情况下都有效，但是在大多数版本的Internet Explorer中都会失败。在那种情况下，推荐的方法是改用纯DOM方法。通过将target元素包装在jQuery对象中，可以处理这些边缘情况，并在所有受支持的浏览器中实现预期的结果：

xxxxxxxxxx
// 使用jQuery设置内部HTML.
var target = document.getElementById( "target" );
$( target ).html( "<td>Hello <b>World</b>!</td>" );

便利性 还有很多常见的DOM操作用例，很难用纯DOM方法完成。例如，在元素newElement之后插入存储的target元素需要相当冗长的DOM方法：

xxxxxxxxxx
// 使用原生DOM API插入一个又一个新元素.
var target = document.getElementById( "target" );
var newElement = document.createElement( "div" );
target.parentNode.insertBefore( newElement, target.nextSibling );

通过将target元素包装在jQuery对象中，相同的任务变得更加简单：

// 使用 jQuery 插入一个又一个新元素.
var target = document.getElementById( "target" );
var newElement = document.createElement( "div" );
$( target ).after( newElement );

将元素放入jQuery对象

当使用CSS选择器调用jQuery函数时，它将返回一个jQuery对象，其中包含与该选择器匹配的所有元素。例如，编写：

x
// 选择所有 <h1> 标签.
var headings = $( "h1" );

headings现在包含一个jQuery元素的所有的<h1>标签已经在页面上。这可以通过检查标题的.length属性来验证:

xxxxxxxxxx
// 在页面上查看所有 <h1> 标签的数值.
var headings = $( "h1" );
alert( headings.length );

如果页面上有多个<h1>标签，则此数字将大于一个。如果页面没有<h1>标签，则.length属性将为零。检查.length属性是确保选择器成功匹配一个或多个元素的一种常用方法。

如果目标是仅选择第一个标题元素，则需要执行另一个步骤。有许多方法可以实现此目的，但是最直接的是.eq()函数。

xxxxxxxxxx
// 在页面上选择第一个<h1>元素 (在jQuery对象中)
var headings = $( "h1" );
var firstHeading = headings.eq( 0 );

现在firstHeading是一个仅包含<h1>页面上第一个元素的jQuery对象。并且由于firstHeading是jQuery对象，因此具有诸如.html()和的有用方法.after()。jQuery还有一个名为的方法.get()，它提供了相关的功能。它没有返回jQuery包装的DOM元素，而是返回DOM元素本身。

xxxxxxxxxx
// 只选择页面上的第一个<h1>元素.
var firstHeadingElem = $( "h1" ).get( 0 );

另外，由于jQuery对象是“类似数组的”，因此它通过方括号支持数组下标：

// 只选择页面上的第一个<h1>元素 (替代的方法).
var firstHeadingElem = $( "h1" )[ 0 ];

无论哪种情况，都firstHeadingElem包含本机DOM元素。这意味着它具有DOM属性（如.innerHTML和方法）.appendChild()，但没有jQuery方法（如.html()或）.after()。该firstHeadingElem元素很难使用，但是在某些情况下需要使用它。这样的例子之一就是进行比较。

并不是所有jQuery对象都被创建===

关于这种“包装”行为的一个重要细节是，每个被包装的对象都是唯一的。即使对象是用相同的选择器创建的，或者包含对完全相同的DOM元素的引用，也是如此。

xxxxxxxxxx
// 为同一个元素创建两个jQuery对象.
var logo1 = $( "#logo" );
var logo2 = $( "#logo" );

尽管logo1和logo2是以相同的方式创建的（并包装相同的DOM元素），但它们不是同一对象。例如：

xxxxxxxxxx
// 比较 jQuery 对象.
alert( $( "#logo" ) === $( "#logo" ) ); // alerts "false"

但是，两个对象都包含相同的DOM元素。该.get()方法对于测试两个jQuery对象是否具有相同的DOM元素很有用。

x
// 比较 DOM 元素.
var logo1 = $( "#logo" );
var logo1Elem = logo1.get( 0 );
 
var logo2 = $( "#logo" );
var logo2Elem = logo2.get( 0 );
 
alert( logo1Elem === logo2Elem ); // alerts "true"

许多开发人员在包含jQuery对象的变量名前面加上$，以帮助区分。这种做法没有什么神奇的-它只是帮助一些人跟踪不同的变量包含什么。前面的例子可以按照这个约定重写:

xxxxxxxxxx
// 比较 DOM 元素 (具有更易读的变量名). 
var $logo1 = $( "#logo" );
var logo1 = $logo1.get( 0 );
 
var $logo2 = $( "#logo" );
var logo2 = $logo2.get( 0 );
 
alert( logo1 === logo2 ); // alerts "true"

该代码的功能与上面的示例相同，但是阅读起来更加清晰。

无论使用哪种命名约定，区分jQuery对象和本机DOM元素都是非常重要的。本机DOM方法和属性不存在于jQuery对象上，反之亦然。错误消息，例如“ event.target.closest不是函数””和“ TypeError：对象[object Object]没有方法'setAttribute'”，表明存在此常见错误。

jQuery对象不是“实时”的

给定一个jQuery对象，该对象具有页面上的所有段落元素：

xxxxxxxxxx
// 选择页面上的所有<p>元素.
var allParagraphs = $( "p" );

人们可能会期望<p>随着文档中元素的添加和删除，内容会随着时间的推移而增加和缩小。jQuery对象不会以这种方式运行。除非明确修改，否则jQuery对象中包含的元素集将不会更改。这意味着该集合不是“实时”的-它不会随着文档的更改而自动更新。如果自创建jQuery对象以来文档可能已更改，则应通过创建一个新的集合来更新该集合。就像重新运行相同的选择器一样简单：

xxxxxxxxxx
// 更新选择.
allParagraphs = $( "p" );

 

结束

尽管DOM元素提供了创建交互式网页所需的所有功能，但它们可能使您工作起来很麻烦。jQuery对象包装了这些元素，以简化这种体验并使常见任务变得容易。当使用jQuery创建或选择元素时，结果将始终包装在新的jQuery对象中。如果情况需要本机DOM元素，则可以通过.get()方法和/或数组样式下标来访问它们。
