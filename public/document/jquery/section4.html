元素的属性可以包含有关您的应用程序的有用信息，因此能够获取和设置它们很重要。

.attr()方法

.attr()方法同时充当getter和setter。作为setter， .attr()既可以接受键和值，也可以接受包含一个或多个键/值对的对象。

.attr()作为一个setter:

$( "a" ).attr( "href", "allMyHrefsAreTheSameNow.html" );
 
$( "a" ).attr({
    title: "all titles are the same too!",
    href: "somethingNew.html"
});

.attr()作为一个getter:

xxxxxxxxxx
$( "a" ).attr( "href" ); // 返回文档中第一个元素的href

 

选择元素

jQuery的最基本概念是“选择一些元素并对其进行处理”。jQuery支持大多数CSS3选择器以及一些非标准选择器。有关完整的选择器参考，请访问api.jquery.com上的选择器文档。

通过ID选择元素
xxxxxxxxxx
$( "#myId" ); // 注意，每个页面的id必须是唯一的。
通过类名称选择元素
$( ".myClass" );
通过属性选择元素
$( "input[name='first_name']" );
通过复合CSS选择器选择元素
$( "#contents ul.people li" );
用逗号分隔的选择器列表选择元素
$( "div.myClass, ul.people" );
伪类选择器
$( "a.external:first" );
$( "tr:odd" );
 
// Select all input-like elements in a form (more on this below).
$( "#myForm :input" );
$( "div:visible" );
 
// All except the first three divs.
$( "div:gt(2)" );
 
// All currently animated divs.
$( "div:animated" );

注意：使用:visible和:hidden伪选择器时，jQuery将测试元素的实际可见性，而不是CSSvisibility或display属性。jQuery会查看该元素在页面上的物理高度和宽度是否都大于零。

但是，此测试不适用于<tr>元素。在<tr>jQuery的情况下，请检查CSSdisplay属性，如果元素的display属性设置为，则认为该元素是隐藏的none。

即使未添加到DOM的元素将对其产生影响，但始终会被视为隐藏元素。

选择 选择器

选择好的选择器是提高JavaScript性能的一种方法。太具体可能不是件好事。一个选择器，例如#myTable thead tr th.special是矫枉过正，如果选择器是#myTable th.special将完成这项工作。

我的选择中是否包含任何元素？

做出选择后，您通常会想知道是否有任何需要处理的内容。一个常见的错误是使用：

x
// 不起作用!
if ( $( "div.foo" ) ) {
    ...
}

这行不通。使用进行选择时$()，总是返回一个对象，而对象的求值始终为true。即使选择不包含任何元素，该if语句内的代码仍将运行。

确定是否存在任何元素的最佳方法是测试所选内容的.length属性，该属性会告诉您选择了多少个元素。如果答案为0，则该.length属性false在用作布尔值时的计算结果为：

xxxxxxxxxx
// 测试所选内容是否包含元素。
if ( $( "div.foo" ).length ) {
    ...
}
保存选择

jQuery不会为您缓存元素。如果做出了可能需要再次进行选择的选择，则应将选择保存在变量中，而不要重复进行选择。

var divs = $( "div" );

将选择存储在变量中之后，就可以像在原始选择上调用jQuery方法一样在变量上调用jQuery方法。

选择仅获取做出选择时页面上的元素。如果稍后将元素添加到页面，则必须重复选择或以其他方式将其添加到存储在变量中的选择中。DOM更改时，存储的选择不会神奇地更新。

精炼和过滤选择

有时候，选择中包含的内容超出了您的期望。jQuery提供了几种优化和过滤选择的方法。

x
// 精炼的选择。
$( "div.foo" ).has( "p" );         // div.foo 元素包含 <p> 标签
$( "h1" ).not( ".bar" );           // h1 元素 不存在一个 bar 类
$( "ul li" ).filter( ".current" ); // 当前类别的无序列表项
$( "ul li" ).first();              // 第一个无序列表项
$( "ul li" ).eq( 5 );              // 第六个
选择表单元素

jQuery提供了几个伪选择器，可以帮助查找表单中的元素。这些功能特别有用，因为使用标准CSS选择器可能很难根据表单元素的状态或类型来区分表单元素。

:checked

不要与：checkbox混淆，它:checked以选中的复选框为目标，但是请记住，此选择器也适用于选中的单选按钮和<select>元素（<select>仅对于元素，请使用:selected选择器）：

$( "form :checked" );

:checked伪选择器在与复选框、单选按钮和选择一起使用时有效。

:disabled

使用:disabled伪选择器将<input>具有以下disabled属性的任何元素作为目标：

$( "form :disabled" );

为了使用来获得最佳性能:disabled，请首先使用标准jQuery选择器选择元素，然后使用.filter( ":disabled" )，或者在伪选择器之前加上标签名称或其他选择器。

:enabled

基本上与:disabled伪选择器相反，:enabled伪选择器针对的是任何没有disabled属性的元素:

$( "form :enabled" );

为了使用来获得最佳性能:enabled，请首先使用标准jQuery选择器选择元素，然后使用.filter( ":enabled" )，或者在伪选择器之前加上标签名称或其他选择器。

:input

使用:input选择器选择所有<input>，<textarea>，<select>，和<button>元素：

$( "form :input" );

:selected

使用:selected伪选择器将<option>元素中的所有选定项目作为目标：

$( "form :selected" );

为了使用来获得最佳性能:selected，请首先使用标准jQuery选择器选择元素，然后使用.filter( ":selected" )，或者在伪选择器之前加上标签名称或其他选择器。

Getters & Setters

某些jQuery方法可用于分配或读取所选内容的某些值。当使用值作为参数调用该方法时，它被称为设置器，因为它设置（或分配）了该值。当不带任何参数的方法被调用时，它获取（或读取）元素的值。setter会影响选择中的所有元素，而getter只会为选择中的第一个元素返回请求的值，除外.text()，会检索所有元素的值。

xxxxxxxxxx
// .html()方法将所有h1元素的html设置为"hello world"
$( "h1" ).html( "hello world" );
// .html()方法返回第一个h1元素的html
$( "h1" ).html();
// > "hello world"

设置器返回一个jQuery对象，使您可以继续在选择时调用jQuery方法。getter返回他们被要求获取的任何内容，因此您不能继续对getter返回的值调用jQuery方法。

xxxxxxxxxx
// 在调用getter之后试图调用jQuery方法
// 这将会不起作用:
$( "h1" ).html().addClass( "test" );

链式

如果在选择上调用方法，并且该方法返回jQuery对象，则可以继续在对象上调用jQuery方法，而无需暂停分号。这种做法称为“链接”：

$( "#content" ).find( "h3" ).eq( 2 ).html( "new text for the third h3!" );

这将有助于代码的可读性，以打破多行代码的界限：

$( "#content" )
    .find( "h3" )
    .eq( 2 )
    .html( "new text for the third h3!" );

jQuery还提供了.end()一种方法，可以在链的中间更改选择时返回到原始选择：

$( "#content" )
    .find( "h3" )
    .eq( 2 )
        .html( "new text for the third h3!" )
        .end() // Restores the selection to all h3s in #content
    .eq( 0 )
        .html( "new text for the first h3!" );

链接是非常强大的，自从它被jQuery普及以来，许多库都采用了这个特性。然而，在使用时必须小心谨慎——大量的链接可能会使代码极其难以修改或调试。对于一条链条应该有多长，并没有硬性规定——只要知道它很容易被冲昏头脑就行了。
