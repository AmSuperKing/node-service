(1)使用jQuery Core

$ VS $()

到目前为止，我们一直在处理在jQuery对象上调用的方法。例如：

$( "h1" ).remove();

如上所述，大多数jQuery方法都是在jQuery对象上调用的；这些方法被称为$.fn名称空间或“ jQuery原型”的一部分，最好将其视为jQuery对象方法。

然而，有几个方法并不作用于选择;这些方法被称为jQuery名称空间的一部分，最好将其视为jQuery核心方法。

这种区别可能会使新的jQuery用户感到难以置信。这是您需要记住的内容：

  • 在jQuery选择上调用的方法位于$.fn名称空间中，并以形式自动接收并返回选择this。
  • $命名空间中的方法通常是实用程序类型的方法，不适用于选择项。它们不会自动传递任何参数，并且它们的返回值会有所不同。

在某些情况下，对象方法和核心方法具有相同的名称，例如$.each()和.each()。在阅读您正在研究的正确方法的文档时，一定要非常小心。

在本指南中，如果可以在jQuery选择中调用方法，则仅以其名称来引用它：.each()。如果它是一个实用程序方法（即未在选择中调用的方法），我们将在jQuery名称空间中将其显式引用为方法：$.each()。

 

$(document).ready()

在文档“就绪”之前，无法安全地操纵页面。jQuery为您检测到这种就绪状态。$( document ).ready()仅在页面文档对象模型（DOM）准备好执行JavaScript代码之后，内部代码才会运行。$( window ).on( "load", function() { ... })一旦整个页面（图像或iframe）（而不仅仅是DOM）都准备好了，其中包含的代码就会运行。

xxxxxxxxxx
// A $( document ).ready() block.
$( document ).ready(function() {
    console.log( "ready!" );
});

有经验的开发人员有时使用简写$()的$( document ).ready()。如果您正在编写不熟悉jQuery的人可能会看到的代码，则最好使用长格式。

// Shorthand for $( document ).ready()
$(function() {
    console.log( "ready!" );
});

您也可以将命名函数传递给$( document ).ready()而不是传递匿名函数。

// Passing a named function instead of an anonymous function.
 
function readyFn( jQuery ) {
    // Code to run when the document is ready.
}
 
$( document ).ready( readyFn );
// or:
$( window ).on( "load", readyFn );

下面的示例显示$( document ).ready()和$( window ).on( "load" )正在运行。该代码尝试在<iframe>中加载网站网址，并检查两个事件：

<html>
<head>
    <script src="https://code.jquery.com/jquery-1.9.1.min.js"></script>
    <script>
    $( document ).ready(function() {
        console.log( "document loaded" );
    });
 
    $( window ).on( "load", function() {
        console.log( "window loaded" );
    });
    </script>
</head>
<body>
    <iframe src="http://techcrunch.com"></iframe>
</body>
</html>

 

避免与其他库的冲突

jQuery库及其几乎所有插件都包含在jQuery名称空间中。通常，全局对象也存储在jQuery名称空间内，因此，您不应在jQuery和任何其他库（例如prototype.js，MooTools或YUI）之间发生冲突。

也就是说，有一个警告：默认情况下，jQuery$用作的快捷方式jQuery。因此，如果您正在使用使用该$变量的另一个JavaScript库，则可能会与jQuery发生冲突。为了避免这些冲突，您需要在将jQuery加载到页面之后以及尝试在页面中使用jQuery之前，立即将jQuery置于无冲突模式。

将jQuery置于无冲突模式

当您将jQuery置于无冲突模式时，可以选择分配一个新的变量名来替换$别名。

<!-- Putting jQuery into no-conflict mode. -->
<script src="prototype.js"></script>
<script src="jquery.js"></script>
<script>
 
var $j = jQuery.noConflict();
// $j is now an alias to the jQuery function; creating the new alias is optional.
 
$j(document).ready(function() {
    $j( "div" ).hide();
});
 
// The $ variable now has the prototype meaning, which is a shortcut for
// document.getElementById(). mainDiv below is a DOM element, not a jQuery object.
window.onload = function() {
    var mainDiv = $( "main" );
}
 
</script>

在上面的代码中，$将会恢复为原始库中的含义。您仍然可以在应用程序的其余部分中使用完整的函数名jQuery以及新的别名$j。新别名可以命名为任何你想：jq，$J，awesomeQuery等。

最后，如果您不想定义完整jQuery函数名的另一种替代方法（您真的很喜欢使用$并且不关心使用其他库的$方法），那么您还可以尝试另一种方法：只需将$as添加为参数传递给您的jQuery( document ).ready()函数。在您仍然希望获得真正简洁的jQuery代码的好处，但又不想与其他库发生冲突的情况下，最常使用此方法。

<!-- Another way to put jQuery into no-conflict mode. -->
<script src="prototype.js"></script>
<script src="jquery.js"></script>
<script>
 
jQuery.noConflict();
 
jQuery( document ).ready(function( $ ) {
    // You can use the locally-scoped $ in here as an alias to jQuery.
    $( "div" ).hide();
});
 
// The $ variable in the global scope has the prototype.js meaning.
window.onload = function(){
    var mainDiv = $( "main" );
}
 
</script>

考虑到为了实现完全的兼容性需要更改的代码较少，这可能是大多数代码的理想解决方案。

在其他库之前包含jQuery

上面的代码片段依赖于在prototype.js加载之后加载jQuery。如果在其他库之前包含jQuery，那么在使用jQuery做一些工作时可以使用jQuery，但是$将具有其他库中定义的含义。没有必要通过调用jQuery.noConflict()来放弃$别名。

<!-- Loading jQuery before other libraries. -->
<script src="jquery.js"></script>
<script src="prototype.js"></script>
<script>
 
// Use full jQuery function name to reference jQuery.
jQuery( document ).ready(function() {
    jQuery( "div" ).hide();
});
 
// Use the $ variable as defined in prototype.js
window.onload = function() {
    var mainDiv = $( "main" );
};
 
</script>

jQuery函数引用方法总结

下面是当另一个库的存在导致对该$变量的使用产生冲突时，可以引用jQuery函数的方式的概述：

创建一个新别名

该jQuery.noConflict()方法返回对jQuery函数的引用，因此您可以在所需的任何变量中捕获它：

<script src="prototype.js"></script>
<script src="jquery.js"></script>
<script>
 
// Give $ back to prototype.js; create new alias to jQuery.
var $jq = jQuery.noConflict();
 
</script>
使用立即调用的函数表达式

您可以继续使用标准的$，将代码包装在立即调用的函数表达式中;这也是一个标准的jQuery插件创作模式，在这里作者不能知道是否有其他库将接管$。

<!-- Using the $ inside an immediately-invoked function expression. -->
<script src="prototype.js"></script>
<script src="jquery.js"></script>
<script>
 
jQuery.noConflict();
 
(function( $ ) {
    // Your jQuery code here, using the $
})( jQuery );
 
</script>

请注意，如果使用此技术，则将无法在立即调用的函数内使用prototype.js方法。$将是对jQuery的引用，而不是prototype.js。

使用传递给jQuery( document ).ready()函数的参数
<script src="jquery.js"></script>
<script src="prototype.js"></script>
<script>
 
jQuery(document).ready(function( $ ) {
    // Your jQuery code here, using $ to refer to jQuery.
});
 
</script>

或者为DOM ready函数使用更简洁的语法：

<script src="jquery.js"></script>
<script src="prototype.js"></script>
<script>
 
jQuery(function($){
    // Your jQuery code here, using the $
});
 
</script>
